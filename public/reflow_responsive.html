Reflow and Responsive Design Issues

WCAG 2.1 Success Criterion 1.4.10 Reflow (Level AA): Content can be presented without loss of information or functionality, and without requiring scrolling in two dimensions.

This page demonstrates content that doesn't properly reflow when zoomed to 400% or viewed on small screens.

Fixed Width Container

This container has a fixed width of 800px

This content doesn't reflow properly when the viewport is narrower than its fixed width, forcing users to scroll horizontally. This creates accessibility barriers, especially for users with low vision who need to zoom the page.

According to WCAG 2.1 Success Criterion 1.4.10, content should be viewable and operable when the viewport width is 320 CSS pixels without requiring horizontal scrolling.

Issue: Fixed-width containers that don't adapt to viewport size require horizontal scrolling.

Fixed Grid Layout

Item 1: This grid uses fixed column widths and doesn't collapse on small viewports.
Item 2: When users zoom to 400%, this layout will require horizontal scrolling.
Item 3: The total width exceeds the viewport width on mobile devices.

Issue: Grid layout with fixed column widths doesn't reflow on small screens or when zoomed.
