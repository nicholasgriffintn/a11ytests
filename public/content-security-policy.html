This page is used to test Content Security Policy (CSP) HTTP header detection. The page itself has no accessibility issues, but the HTTP header may be tested by a11y tools.

About Content Security Policy

Content Security Policy (CSP) is an added layer of security that helps to detect and mitigate certain types of attacks, including Cross-Site Scripting (XSS) and data injection attacks.

CSP headers can control which resources are allowed to load on a page, including:

  • Scripts
  • Stylesheets
  • Images
  • Fonts
  • Media (audio/video)
  • Objects
  • Frames

Example CSP Headers

Content-Security-Policy: default-src 'self'; script-src 'self'

This restricts loading of all resources to the same origin as the page.
